ABOUT

Our vision is to deliver a decentralized database layer to unify access to data. Xuma’s groundbreaking technology will offer fundamental yet elusive features such as fault tolerance, censorship resistant, DDOS-resistant, and zero-downtime. Additionally, with the use of blockchain technologies, Xuma feature key technologies that make Xuma’s solution ready to use out of the box.

Decentralized Database

A key technology to encrypt, store, retrieve and distribute data across Xuma’s global database network.

Proof of Stake

Xuma Masternode and Staking technologies serves as the core engine that powers the network. The two technologies combine to process current and future tasks in our decentralized networks. In return for their participation, the network will compensate for work provided in the form of XMX coins.

Smart Contracts

This feature controls the system and allows consumers to define rules based on our market contract. Our smart contract will be able to work across blockchains and reduce if not eliminate data silos prevalent in today’s systems.

SPECS

Xuma (XMX) is a hybrid and PoW/PoS system.
  • Proof of Work Algorithm: Quark
  • Premine: (#1 Block) 120,001 XUMA (0.21%) for 12 Masternodes to stabilize the network
  • Proof of Work Blocks: 2 - 299,999
  • Proof of Stake Blocks: Starting from 300,000
  • Block Time: 60 Seconds
  • Proof of Work Max Coin Supply: 41,249,650
  • Proof of Work ending: Approximately 146 days
  • Masternode collateral: 10,000 XUMA
  • Maturity: 120 Confirmations
  • RPC Port: 19643
  • Port: 19777

INSTRUCTIONS

Setting up swap space on a Ubuntu VPS
$ sudo fallocate -l 4G /swap
$ sudo chmod 600 /swap
$ sudo mkswap /swap
$ sudo swapon /swap
$ echo '/swap none swap sw 0 0' | sudo tee -a /etc/fstab
Compiling and running xumad on Ubuntu:
$ add-apt-repository ppa:bitcoin/bitcoin -y
$ apt-get update
$ apt-get upgrade -y
$ apt-get autoremove -y
$ apt-get install git build-essential libtool autotools-dev autoconf pkg-config libssl-dev libboost-all-dev libdb4.8-dev libdb4.8++-dev libminiupnpc-dev -y
$ mkdir ~/source && cd ~/source && git clone https://github.com/xumacoin/xuma-core.git
$ cd ~/source/xuma-core && ./autogen.sh && ./configure
$ make all install
$ xumad
$ xuma-cli getinfo
$ xuma-cli getbalance
$ xuma-cli getaccountaddress ""
Sample masternode xuma.conf file:
masternode=1
masternodeprivkey=<xuma-cli masternode genkey>
rpcuser=xumarpc
rpcpassword=create_a_random_password_here
staking=0
daemon=1
server=1
listen=1
txindex=1
Setting up a hot masternode:
Get the default wallet address that has a 0 balance: $ xuma-cli getaccountaddress ""
Or create a new wallet address that has a 0 balance: $ xuma-cli getaccountaddress "example"
Send 10,000 XMX to that address from your primary wallet: $ xuma-cli sendtoaddress XLd5zb7KJewkBNH7xb94r9WHUVrxKiXLcS 10000
Allow 15 confirmations and start the masternode: $ xuma-cli masternode start
